Idea to PRD to Prototype

Turn a raw product idea into a working, clickable prototype

Core skills: Prompting  ·  PRD structure  ·  Product thinking  ·  Prototyping

Every product starts somewhere, usually a vague idea with a lot of potential and not enough clarity. Project 1 is about closing that gap in one week. You'll take an idea you genuinely care about, work with Claude Code to shape it into a structured PRD, and then go further, turning that PRD into a real, clickable prototype before the week is out.

The reason we go all the way to prototype in Week 1 is important: a working prototype changes the conversation. It replaces "here's what I'm thinking" with "here's what it looks like." That shift from description to demonstration is one of the most powerful things Claude Code enables.

 

Phase 1 — From Idea to PRD

Write a rough one-paragraph description of your product idea in plain English. Don't worry about format. Just explain what it is, who it's for, and the problem it solves. Use Claude chat interface to generate the following:

Problem Statement

Target Users

Key Features

Success Metrics

What your product is NOT


Use Claude Code's PRD generator to generate a PRD. Claude Code will think, plan, identify gaps, and structure your thinking into a complete PRD with all standard sections.

Review every section critically. Where is it vague? Where does a feature lack a clear "why"? Mark everything that needs more thought.

Refine in conversation with Claude Code, ask it to challenge assumptions, suggest missing edge cases, rewrite weak sections. Iterate until the PRD feels like something you'd hand to an engineering team with confidence.

 

A Solid PRD Covers

Problem statement: One clear paragraph — what's broken, and for whom

Target users: Specific and describable — not "everyone" or "businesses"

Core features: Prioritized as must-have, should-have, nice-to-have

Out of scope: Explicitly listed — this protects you from scope creep in later projects

Success metrics: Named, measurable — how will you actually know this worked

 

Phase 2 — From PRD to Prototype

Once your PRD is solid, you'll use it as the brief for Claude Code to build from. Select one must-have feature from your PRD — the one that best demonstrates the core value of your product — and build a working prototype of it.

Give Claude Code your PRD and ask it to identify the single feature that best demonstrates the product's core value proposition.

Ask Claude Code to build a working prototype of that feature including a functional UI, real interactions, and no placeholder screens.

Review it like a user, not a builder. Does it communicate the idea without explanation? Note anything that needs adjustment and refine in the same session.

 

📌  Note: The prototype doesn't have to be complete. It has to be real enough that someone who's never seen your idea could look at it and immediately understand what the product does.

📦  Deliverable: A complete refined PRD + a working clickable prototype of your core feature.
